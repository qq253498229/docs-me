附录D. 扩展

以下是发布时已确立的一些扩展的列表：

  • [RFC9068]: JSON Web Token (JWT) Profile for OAuth 2.0 Access Tokens

    • 本规范定义了以JSON Web Token (JWT)格式颁发OAuth访问令牌的配置文件。

  • [RFC8628]: OAuth 2.0 Device Authorization Grant

    • 设备授权许可（以前称为设备流）是一种扩展，它使没有浏览器或输入能力有限的设备能够获得访问令牌。 这通常被智能电视应用程序或可以向流媒体视频服务流式传输视频的硬件设备（如硬件视频编码器）所使用。

  • [RFC8414]: Authorization Server Metadata

    • 授权服务器元数据（也称为OAuth发现）定义了一个端点，客户端可以使用该端点来查找与特定OAuth服务器交互所需的信息，例如授权和令牌端点的位置以及支持的授权类型。

  • [RFC8707]: Resource Indicators

    • 提供了一种方式，使客户端能够向授权服务器明确指示它打算在哪里使用它正在请求的访问令牌。

  • [RFC7591]: Dynamic Client Registration

    • 动态客户端注册提供了一种机制，用于通过授权服务器以编程方式注册客户端。

  • [RFC9449]: Demonstrating Proof of Possession (DPoP)

    • DPoP描述了一种机制，该机制通过应用程序级别的所有权证明机制将令牌绑定到它们被颁发的客户端，并在发出请求时在HTTP头中提供该绑定的证明。

  • [RFC8705]: Mutual TLS

    • Mutual TLS描述了一种机制，该机制通过TLS证书认证将令牌绑定到它们被颁发的客户端，以及一种客户端认证机制。

  • [RFC7662]: Token Introspection

    • 令牌内省扩展定义了一种机制，使资源服务器能够获得有关访问令牌的信息。

  • [RFC7009]: Token Revocation

    • 令牌撤销扩展定义了一种机制，使客户端能够向授权服务器指示某个访问令牌不再需要。

  • [RFC9126]: Pushed Authorization Requests

    • 推送授权请求扩展描述了一种从后通道启动OAuth流程的技术，为构建复杂的授权请求提供了更好的安全性和更大的灵活性。

  • [RFC9207]: Authorization Server Issuer Identification

    • 授权响应中的iss参数指示授权服务器的身份，以防止在客户端发生混淆攻击。

  • [RFC9396]: Rich Authorization Requests

    • 丰富授权请求指定了一个新的参数authorization_details，该参数用于在OAuth授权请求中携带细粒度的授权数据。

  • [RFC9449]: Demonstrating Proof of Possession (DPoP)

    • DPoP描述了一种机制，该机制通过应用程序级别的所有权证明机制对发送者进行约束，从而限制OAuth 2.0令牌的使用。

  • [RFC9470]: Step-Up Authentication Challenge Protocol

    • Step-Up Auth描述了一种机制，资源服务器可以使用该机制向客户端发出信号，表明与当前请求的访问令牌相关联的认证事件不符合其认证要求。
