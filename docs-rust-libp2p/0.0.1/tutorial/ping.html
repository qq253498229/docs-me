Ping教程 - rust-libp2p入门

本教程旨在为新手提供一个实践性的概览，介绍如何使用 Rust 的 libp2p 实现。 对于 Rust 的新手来说，可能希望先从 Rust 本身开始，然后再深入到网络的乐趣中。 这个库大量使用了异步 Rust。如果你不熟悉这个概念，Rust 的异步书籍 应该会很有帮助。 对于 libp2p 的新手来说，可能更愿意先在 libp2p.io 上获得一个总体概览，尽管本教程不需要 libp2p 的知识。

我们将构建一个小型的 ping 克隆程序，向一个对等节点发送 ping，期待收到 pong 作为响应。

脚手架

让我们从以下步骤开始：

  1. 更新到最新的 Rust 工具链，例如：rustup update

  2. 创建一个新的 crate：cargo init rust-libp2p-tutorial

  3. 在 Cargo.toml 文件中添加 libp2p 以及 futures 作为依赖项。当前的 crate 版本可以在 crates.io 上找到。我们还将包含带有“attributes”特性的 async-std，以允许 async main。在撰写本文时我们有：

[package]
    name = "rust-libp2p-tutorial"
    version = "0.1.0"
    edition = "2021"

[dependencies]
    libp2p = { version = "0.52", features = ["tcp", "tls", "dns", "async-std", "noise", "yamux", "websocket", "ping", "macros"] }
    futures = "0.3.21"
    async-std = { version = "1.12.0", features = ["attributes"] }
    tracing-subscriber = { version = "0.3", features = ["env-filter"] }

网络身份

随着所有脚手架的到位，我们可以深入到 libp2p 的具体内容。 首先，我们需要在 async fn main() 中为我们的本地节点创建一个网络身份，并用一个属性注解，以允许 main 是 async 的。 在 libp2p 中，身份通过一对公钥/私钥来处理。节点通过它们的 PeerId 来识别彼此，这是从它们的公钥派生出来的。现在，用以下内容替换 main.rs 中的内容：

use std::error::Error;
use tracing_subscriber::EnvFilter;

#[async_std::main]
async fn main() -> Result<(), Box<dyn Error>> {
    tracing_subscriber::fmt().with_env_filter(EnvFilter::from_default_env()).init();

    let mut swarm = libp2p::SwarmBuilder::with_new_identity();

    Ok(())
}

继续构建并运行上述代码：cargo run。到目前为止还没有发生什么。

传输

接下来我们需要构建一个传输。libp2p 中的每个传输都提供加密流。例如，结合 TCP 来建立连接，TLS 来加密这些连接，以及 Yamux 在一个连接上运行一个或多个流。另一个 libp2p 传输是 QUIC，它提供了开箱即用的加密流。我们现在将坚持使用 TCP。这些传输都实现了 Transport trait。

网络行为

现在我们来看 rust-libp2p 的另一个核心特性：NetworkBehaviour。虽然之前介绍的 Transport trait 定义了如何在网络中发送字节，NetworkBehaviour 定义了在网络中发送哪些字节以及发送给谁。

为了更具体地说明这一点，让我们看看 NetworkBehaviour trait 的一个简单实现：ping::Behaviour。 正如你可能已经猜到的，类似于旧的 ICMP ping 网络工具，libp2p ping::Behaviour 向一个对等节点发送 ping，并期望收到 pong 作为回应。 ping::Behaviour 不关心 ping 和 pong 消息是如何在网络上发送的，无论是通过 TCP 发送，是否通过 noise 加密，还是仅仅是 明文。 它只关心在网络中发送哪些消息以及发送给谁。

Transport 和 NetworkBehaviour 这两个特性使我们能够清晰地分离如何发送字节与发送 _哪些 字节以及发送给 谁。

考虑到上述内容，让我们扩展我们的例子，在最后创建一个 ping::Behaviour：

use libp2p::ping;
use tracing_subscriber::EnvFilter;
use std::error::Error;

#[async_std::main]
async fn main() -> Result<(), Box<dyn Error>> {
    tracing_subscriber::fmt().with_env_filter(EnvFilter::from_default_env()).init();

    let mut swarm = libp2p::SwarmBuilder::with_new_identity()
        .with_async_std()
        .with_tcp(
            libp2p::tcp::Config::default(),
            libp2p::tls::Config::new,
            libp2p::yamux::Config::default,
        )?
        .with_behaviour(|_| ping::Behaviour::default())?;

    Ok(())
}

Swarm

现在我们有了传输（Transport）和网络行为（NetworkBehaviour），我们可以构建一个连接两者的 Swarm，允许两者都取得进展。 简单来说，Swarm 推动 Transport 和 NetworkBehaviour 向前发展，将命令从 NetworkBehaviour 传递到 Transport，并将事件从 Transport 传递到 NetworkBehaviour。

use libp2p::ping;
use std::error::Error;
use tracing_subscriber::EnvFilter;

#[async_std::main]
async fn main() -> Result<(), Box<dyn Error>> {
    tracing_subscriber::fmt().with_env_filter(EnvFilter::from_default_env()).init();

    let mut swarm = libp2p::SwarmBuilder::with_new_identity()
        .with_async_std()
        .with_tcp(
            libp2p::tcp::Config::default(),
            libp2p::tls::Config::new,
            libp2p::yamux::Config::default,
        )?
        .with_behaviour(|_| ping::Behaviour::default())?
        .build();

    Ok(())
}

空闲连接超时

现在，对于这个特定的例子，我们需要设置空闲连接超时。否则，连接将立即关闭。

你是否也需要在你的应用程序中设置这个，取决于你的用例。通常情况下，如果连接被某个协议“使用”，则会保持连接。然而，ping协议只是一个“辅助”类型的协议。因此，如果没有其他行为到位，我们将无法观察到ping。

use libp2p::ping;
use std::error::Error;
use std::time::Duration;
use tracing_subscriber::EnvFilter;

#[async_std::main]
async fn main() -> Result<(), Box<dyn Error>> {
    tracing_subscriber::fmt().with_env_filter(EnvFilter::from_default_env()).init();

    let mut swarm = libp2p::SwarmBuilder::with_new_identity()
        .with_async_std()
        .with_tcp(
            libp2p::tcp::Config::default(),
            libp2p::tls::Config::new,
            libp2p::yamux::Config::default,
        )?
        .with_behaviour(|_| ping::Behaviour::default())?
        .with_swarm_config(|cfg| cfg.with_idle_connection_timeout(Duration::from_secs(u64::MAX))) // Allows us to observe pings indefinitely.
        .build();

    Ok(())
}

Multiaddr

有了 Swarm，我们准备监听传入的连接。我们只需要像 std::net::TcpListener::bind 一样向Swarm 传递一个地址。 但我们传递的不是IP地址，而是一个 Multiaddr，这是libp2p中另一个值得一看的核心概念。

Multiaddr 是一个自描述的网络地址和协议栈，用于建立与对等方的连接。 可以在 docs.libp2p.io/concepts/addressing 找到 Multiaddr 的良好介绍，其规范仓库位于 github.com/multiformats/multiaddr。

让我们使我们的本地节点在一个新的套接字上监听。这个套接字同时在多个网络接口上监听。对于每个网络接口，都会创建一个新的监听地址。这些地址可能随着接口的可用或不可用而变化。例如，对于我们的TCP传输，它可能（除其他外）在回环接口（localhost）/ip4/127.0.0.1/tcp/24915 以及本地网络 /ip4/192.168.178.25/tcp/24915 上监听。

此外，如果CLI提供了，让我们指示我们的本地节点拨打远程对等方。

use libp2p::{ping, Multiaddr};
use std::error::Error;
use std::time::Duration;
use tracing_subscriber::EnvFilter;

#[async_std::main]
async fn main() -> Result<(), Box<dyn Error>> {
    tracing_subscriber::fmt().with_env_filter(EnvFilter::from_default_env()).init();

    let mut swarm = libp2p::SwarmBuilder::with_new_identity()
        .with_async_std()
        .with_tcp(
            libp2p::tcp::Config::default(),
            libp2p::tls::Config::new,
            libp2p::yamux::Config::default,
        )?
        .with_behaviour(|_| ping::Behaviour::default())?
        .with_swarm_config(|cfg| cfg.with_idle_connection_timeout(Duration::from_secs(u64::MAX))) // Allows us to observe pings indefinitely..
        .build();

    // 告诉集群监听所有接口和一个随机的、操作系统分配的端口。
    swarm.listen_on("/ip4/0.0.0.0/tcp/0".parse()?)?;

    // 拨打由第二个命令行参数给出的多地址标识的对等体（如果有的话）。
    if let Some(addr) = std::env::args().nth(1) {
        let remote: Multiaddr = addr.parse()?;
        swarm.dial(remote)?;
        println!("Dialed {addr}")
    }

    Ok(())
}

持续轮询Swarm

现在一切都已就绪。最后一步是在循环中驱动 Swarm，允许它监听传入的连接，并在CLI上指定地址的情况下建立出站连接。

use futures::prelude::*;
use libp2p::swarm::SwarmEvent;
use libp2p::{ping, Multiaddr};
use std::error::Error;
use std::time::Duration;
use tracing_subscriber::EnvFilter;

#[async_std::main]
async fn main() -> Result<(), Box<dyn Error>> {
    tracing_subscriber::fmt().with_env_filter(EnvFilter::from_default_env()).init();

    let mut swarm = libp2p::SwarmBuilder::with_new_identity()
        .with_async_std()
        .with_tcp(
            libp2p::tcp::Config::default(),
            libp2p::tls::Config::new,
            libp2p::yamux::Config::default,
        )?
        .with_behaviour(|_| ping::Behaviour::default())?
        .with_swarm_config(|cfg| cfg.with_idle_connection_timeout(Duration::from_secs(u64::MAX))) // Allows us to observe pings indefinitely.
        .build();

    // 告诉集群监听所有接口和一个随机的、操作系统分配的端口。
    swarm.listen_on("/ip4/0.0.0.0/tcp/0".parse()?)?;

    // 拨打由第二个命令行参数给出的多地址标识的对等体（如果有的话）。
    if let Some(addr) = std::env::args().nth(1) {
        let remote: Multiaddr = addr.parse()?;
        swarm.dial(remote)?;
        println!("Dialed {addr}")
    }

    loop {
        match swarm.select_next_some().await {
            SwarmEvent::NewListenAddr { address, .. } => println!("Listening on {address:?}"),
            SwarmEvent::Behaviour(event) => println!("{event:?}"),
            _ => {}
        }
    }
}

运行两个节点

为了方便，上述例子也在 examples/ping.rs 中完整实现。因此，你可以从你自己的项目中运行下面的命令，也可以从rust-libp2p仓库的根目录运行。注意，在前者的情况下，你需要忽略 --example ping 参数。

你需要两个终端。在第一个终端窗口运行：

cargo run --example ping

它将打印新的监听地址，例如：

Listening on "/ip4/127.0.0.1/tcp/24915"
Listening on "/ip4/192.168.178.25/tcp/24915"
Listening on "/ip4/172.17.0.1/tcp/24915"
Listening on "/ip6/::1/tcp/24915"

在第二个终端窗口，启动示例的新实例：

cargo run --example ping -- /ip4/127.0.0.1/tcp/24915

注意：最后的 Multiaddr 是之前在终端窗口一中打印的 Multiaddr 之一。两个对等方必须在同一网络中，与地址相关联。在我们的情况下，可以使用任何打印的地址，因为两个对等方都在同一设备上运行。

这两个节点将建立连接，并且每15秒互相发送ping和pong消息。
