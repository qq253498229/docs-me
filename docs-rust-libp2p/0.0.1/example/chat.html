聊天应用程序示例

描述

一个展示libp2p和gossipsub协议结合mDNS用于发现聊天对象的基本聊天应用。它展示了节点如何连接、使用mDNS发现彼此，并进行实时聊天会话。

用法

  • 1.使用两个终端窗口，启动两个实例，在每个窗口中输入以下内容：

cargo run

  • 2.相互mDNS发现可能需要几秒钟。当每个节点发现对方时，它会打印一条消息，如下：

mDNS discovered a new peer: {peerId}

  • 3.输入一条消息并按回车键：消息被发送并在另一个终端打印。

  • 4.使用 Ctrl-c 关闭。您可以打开更多的终端窗口，并使用上面的相同行添加更多的节点。

当一个新节点通过mDNS被发现时，它可以加入对话，所有节点都会接收到该节点发送的消息。 如果一个参与者使用 Ctrl-c 或任何其他方法退出应用程序，剩余的节点将接收到一个mDNS过期事件，并从它们已知节点列表中移除过期的节点。

结论

这个聊天应用展示了如何使用libp2p和gossipsub协议构建一个去中心化的聊天系统。 通过利用mDNS进行节点发现，用户可以轻松地与其他节点连接并进行实时对话。 这个示例为使用libp2p开发更复杂的聊天应用和探索去中心化通信的能力提供了一个起点。

代码

Cargo.toml

[package]
name = "chat-example"
version = "0.1.0"
edition = "2021"
publish = false
license = "MIT"

[package.metadata.release]
release = false

[dependencies]
tokio = { workspace = true, features = ["full"] }
async-trait = "0.1"
futures = { workspace = true }
libp2p = { path = "../../libp2p", features = [ "tokio", "gossipsub", "mdns", "noise", "macros", "tcp", "yamux", "quic"] }
tracing = { workspace = true }
tracing-subscriber = { workspace = true, features = ["env-filter"] }

[lints]
workspace = true

main.rs

#![doc = include_str!("../README.md")]

use futures::stream::StreamExt;
use libp2p::{gossipsub, mdns, noise, swarm::NetworkBehaviour, swarm::SwarmEvent, tcp, yamux};
use std::collections::hash_map::DefaultHasher;
use std::error::Error;
use std::hash::{Hash, Hasher};
use std::time::Duration;
use tokio::{io, io::AsyncBufReadExt, select};
use tracing_subscriber::EnvFilter;

// 我们创建了一个自定义的网络行为，它结合了Gossipsub和Mdns。
#[derive(NetworkBehaviour)]
struct MyBehaviour {
    gossipsub: gossipsub::Behaviour,
    mdns: mdns::tokio::Behaviour,
}

#[tokio::main]
async fn main() -> Result<(), Box<dyn Error>> {
    let _ = tracing_subscriber::fmt()
        .with_env_filter(EnvFilter::from_default_env())
        .try_init();

    let mut swarm = libp2p::SwarmBuilder::with_new_identity()
        .with_tokio()
        .with_tcp(
            tcp::Config::default(),
            noise::Config::new,
            yamux::Config::default,
        )?
        .with_quic()
        .with_behaviour(|key| {
            // 对于内容地址消息，我们可以取消息的哈希值并将其用作ID。
            let message_id_fn = |message: &gossipsub::Message| {
                let mut s = DefaultHasher::new();
                message.data.hash(&mut s);
                gossipsub::MessageId::from(s.finish().to_string())
            };

            // 设置自定义gossipsub配置
            let gossipsub_config = gossipsub::ConfigBuilder::default()
                .heartbeat_interval(Duration::from_secs(10)) // This is set to aid debugging by not cluttering the log space
                .validation_mode(gossipsub::ValidationMode::Strict) // 设置消息验证模式。默认值是Strict（强制消息签名）
                .message_id_fn(message_id_fn) // 内容地址消息。两条消息之间不会有相同内容。
                .build()
                .map_err(|msg| io::Error::new(io::ErrorKind::Other, msg))?; // 临时hack一下，因为 `build` 没有返回 `std::error::Error`.

            // 建立gossipsub网络行为
            let gossipsub = gossipsub::Behaviour::new(
                gossipsub::MessageAuthenticity::Signed(key.clone()),
                gossipsub_config,
            )?;

            let mdns =
                mdns::tokio::Behaviour::new(mdns::Config::default(), key.public().to_peer_id())?;
            Ok(MyBehaviour { gossipsub, mdns })
        })?
        .with_swarm_config(|c| c.with_idle_connection_timeout(Duration::from_secs(60)))
        .build();

    // 创建一个Gossipsub话题
    let topic = gossipsub::IdentTopic::new("test-net");
    // 订阅我们的主题
    swarm.behaviour_mut().gossipsub.subscribe(&topic)?;

    // 从stdin中读取完整的行
    let mut stdin = io::BufReader::new(io::stdin()).lines();

    // 监听所有接口和操作系统分配的任何端口
    swarm.listen_on("/ip4/0.0.0.0/udp/0/quic-v1".parse()?)?;
    swarm.listen_on("/ip4/0.0.0.0/tcp/0".parse()?)?;

    println!("通过STDIN输入消息，它们将使用Gossipsub发送到连接的对等体");

    // Kick it off
    loop {
        select! {
            Ok(Some(line)) = stdin.next_line() => {
                if let Err(e) = swarm
                    .behaviour_mut().gossipsub
                    .publish(topic.clone(), line.as_bytes()) {
                    println!("发布错误: {e:?}");
                }
            }
            event = swarm.select_next_some() => match event {
                SwarmEvent::Behaviour(MyBehaviourEvent::Mdns(mdns::Event::Discovered(list))) => {
                    for (peer_id, _multiaddr) in list {
                        println!("mDNS发现了一个新的节点: {peer_id}");
                        swarm.behaviour_mut().gossipsub.add_explicit_peer(&peer_id);
                    }
                },
                SwarmEvent::Behaviour(MyBehaviourEvent::Mdns(mdns::Event::Expired(list))) => {
                    for (peer_id, _multiaddr) in list {
                        println!("mDNS发现节点过期: {peer_id}");
                        swarm.behaviour_mut().gossipsub.remove_explicit_peer(&peer_id);
                    }
                },
                SwarmEvent::Behaviour(MyBehaviourEvent::Gossipsub(gossipsub::Event::Message {
                    propagation_source: peer_id,
                    message_id: id,
                    message,
                })) => println!(
                        "收到消息: '{}' id是: {id} 节点为: {peer_id}",
                        String::from_utf8_lossy(&message.data),
                    ),
                SwarmEvent::NewListenAddr { address, .. } => {
                    println!("本地节点正在监听 {address}");
                }
                _ => {}
            }
        }
    }
}
